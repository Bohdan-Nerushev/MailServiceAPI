# 📧 Mail Service API (FastAPI)

Цей проект надає REST API для керування поштовим сервером на базі **Postfix** та **Dovecot**. Система дозволяє створювати користувачів, змінювати паролі, а також надсилати та отримувати електронну пошту через веб-інтерфейс або програмний інтерфейс.

---

## 🚀 1. Команди для експлуатації (на віддаленому сервері)

Для розгортання та запуску проекту на віддаленому сервері (Linux) виконайте наступні кроки:

### Підготовка середовища
```bash
# Клонування репозиторію
git clone https://github.com/Bohdan-Nerushev/MailServiceAPI.git
cd MailServiceAPI

# Створення віртуального середовища
python3 -m venv venv

# Активація середовища
source venv/bin/activate

# Встановлення залежностей
pip install -r requirements.txt
```

### Налаштування змінних оточення
Створіть файл `.env` та заповніть його:
```bash

nano .env
```
**Важливо:** Переконайтеся, що `SUDO_USER_PASSWORD` встановлено, якщо API потребує прав sudo для керування системними користувачами.

### Запуск проекту
Для стабільної роботи в продуктовому середовищі рекомендується використовувати **Gunicorn**:
```bash
# Запуск через Gunicorn з Uvicorn workers
export PYTHONPATH=$PYTHONPATH:$(pwd)
venv/bin/gunicorn app.main:app \
    --workers 4 \
    --worker-class uvicorn.workers.UvicornWorker \
    --bind 0.0.0.0:8090 \
    --daemon
```
*Для перевірки логів використовуйте:* `tail -f app.log`

## 🛠 2. Технології та бібліотеки

Проект побудований з використанням сучасного стеку технологій:

### Core Frameworks & Servers
- **Python 3.10+**: Основна мова розробки.
- **FastAPI**: Швидкий асинхронний веб-фреймворк для створення API.
- **Uvicorn / Gunicorn**: ASGI-сервери для обслуговування запитів.

### Робота з поштою та системою
- **imap-tools**: Бібліотека для зручної роботи з протоколом IMAP (читання листів).
- **aiosmtplib**: Асинхронний клієнт для надсилання пошти через SMTP.
- **email-validator**: Валідація адрес електронної пошти.
- **psutil**: Моніторинг системних ресурсів (CPU, RAM, Disk).

### UI та шаблонізація
- **Jinja2**: Двигун шаблонів для генерації HTML-сторінок інтерфейсу.
- **CSS (Vanilla)**: Стилізація інтерфейсу без використання важких фреймворків.

### Інфраструктура (Зовнішні сервіси)
- **Postfix**: SMTP-сервер для надсилання та отримання пошти.
- **Dovecot**: IMAP/POP3 сервер для зберігання та доступу до листів.
- **SpamAssassin**: Система фільтрації спаму.
- **Procmail**: Засіб для локальної доставки та сортування пошти.

## 🔗 3. Опис ендпоінтів

Всі ендпоінти поділені за категоріями для зручності використання.

### 🍏 Система та Health Check (`/health`)
| Ендпоінт | Метод | Призначення |
| :--- | :--- | :--- |
| `/health` | `GET` | Повертає детальний статус системи, стан сервісів Postfix, Dovecot, SpamAssassin, а також метрики ресурсів сервера. |

### 👤 Керування користувачами (`/users`)
*Ці ендпоінти взаємодіють безпосередньо з системними обліковими записами Linux.*
| Ендпоінт | Метод | Призначення |
| :--- | :--- | :--- |
| `/users/` | `GET` | Отримання списку всіх системних користувачів (UID >= 1000). |
| `/users/` | `POST` | Створення нового системного користувача без доступу до shell (nologin) для поштової скриньки. |
| `/users/{user}/password`| `PUT` | Зміна пароля існуючого користувача. |
| `/users/{user}` | `DELETE` | Видалення користувача та його домашньої директорії (поштової скриньки). |

### 📧 Поштовий API (`/mail`)
| Ендпоінт | Метод | Призначення |
| :--- | :--- | :--- |
| `/mail/send` | `POST` | Надсилання електронного листа через локальний SMTP сервер. |
| `/mail/inbox/{user}` | `GET` | Отримання списку листів з папки INBOX для вказаного користувача. |
| `/mail/message/{uid}` | `GET` | Отримання повного змісту конкретного листа за його UID. |
| `/mail/message/{uid}` | `DELETE` | Перманентне видалення листа. |

### 💻 Веб-інтерфейс (`/ui`)
*Графічний інтерфейс для користувачів (рендер HTML через Jinja2).*
| Ендпоінт | Метод | Призначення |
| :--- | :--- | :--- |
| `/ui/` | `GET` | Головна сторінка (Dashboard). |
| `/ui/login` | `GET/POST` | Сторінка входу та обробка авторизації. |
| `/ui/register` | `GET/POST` | Сторінка реєстрації нових поштових акаунтів. |
| `/ui/inbox` | `GET` | Перегляд вхідних повідомлень (підтримує папки: INBOX, Sent, Trash). |
| `/ui/compose` | `GET/POST` | Форма створення та надсилання листа. |
| `/ui/mail/{uid}` | `GET` | Детальний перегляд листа. |
| `/ui/health` | `GET` | Візуальна панель моніторингу стану сервера. |
| `/ui/logout` | `POST` | Вихід з облікового запису (очищення сесійних cookie). |
